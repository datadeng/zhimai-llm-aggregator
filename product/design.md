## 一、用户体验分析（UX Analysis）

### 1.1 用户首次访问关注点

| 优先级 | 用户关注点 | 设计响应 |
|--------|-----------|---------|
| P0 | 这个网站能帮我找到什么 API？ | 首屏大标题 + 搜索框，30秒内给出答案 |
| P0 | 这些 API 质量如何？价格贵不贵？ | 卡片展示评分/价格/热门标签 |
| P1 | 怎么开始使用？要不要注册？ | 一键跳转官方 + 简洁引导 |
| P1 | 推廣能赚钱吗？佣金多少？ | 首屏底部/悬浮按钮展示佣金入口 |
| P2 | 有没有优惠/活动？ | 横幅/公告栏 |

**首屏设计原则：** 用户进入首页 5 秒内必须看到"我能在这里做什么"。

---

### 1.2 核心用户旅程

```
[访问首页] → [搜索/浏览 API] → [查看详情/比价] → [跳转官网注册/充值] → [成为推广员赚佣金]
     ↓              ↓                   ↓               ↓                    ↓
   陌生用户      目标明确用户        比价决策用户      付费转化用户         推广变现用户
```

**关键转化节点：**

1. **搜索 → 详情**：搜索命中 → 点击卡片 → 详情页
2. **详情 → 跳转**：查看定价/文档 → 点击"去官网"按钮（追踪跳转来源）
3. **用户注册 → 推广员**：注册账号 → 申请成为推广员 → 获取专属链接
4. **推广员成交 → 佣金**：下线付费 → 推广员获得佣金通知 → 申请提现

---

### 1.3 快速找到 API 的设计策略

- **智能搜索**：支持中英文、模糊匹配、按用途/价格/厂商搜索
- **分类导航**：热门分类（文本生成/图像/语音/Embedding/视频）快捷入口
- **筛选排序**：按价格/评分/热度/最新排序
- **热门推荐**：首页展示"本周热榜"、"新人首选"
- **对比功能**：勾选多个 API，一键生成对比表

### 2.4 移动端适配策略

- **响应式断点**：Mobile (<768px) / Tablet (768-1024px) / Desktop (>1024px)
- **底部导航**：移动端固定 Bottom Nav（首页/分类/推广/我的）
- **搜索优化**：移动端搜索框吸顶，随滚动收起/展开
- **卡片单列**：移动端 API 卡片单列展示，减少信息密度
- **悬浮按钮**：右下角固定"推广赚钱"悬浮按钮（高转化入口）
- **手势支持**：左右滑动切换分类标签

## 三、设计风格

### 3.1 参考网站分析

参考 https://guo520.xyz/ 的设计特点：
- 简洁清晰的卡片式布局
- 分类导航清晰
- 强调实用性（直达链接）
- 配色以深色/科技感为主

### 3.2 配色方案

```
主题色（Primary）：#6366F1（靛蓝紫 — 科技感、信任感）
辅助色（Secondary）：#10B981（翠绿 — 佣金/收益正向感）
强调色（Accent）：#F59E0B（琥珀金 — 高亮、优惠、热门）
背景色（Background）：#0F172A（深空蓝 — 沉浸感）
卡片背景：#1E293B（深灰蓝）
文字主色：#F8FAFC（近白）
文字次色：#94A3B8（灰蓝）
边框色：#334155
成功色：#22C55E
警告色：#EF4444
```

### 3.3 字体建议

| 用途 | 字体 | 备选 |
|------|------|------|
| 标题 | Inter / 思源黑体 | Noto Sans SC |
| 正文 | Inter / 思源黑体 | Noto Sans SC |
| 代码/价格 | JetBrains Mono | Fira Code |
| 数字强调 | Tabular Numbers（等宽数字） | — |

**字号规范：**
- H1: 32px / H2: 24px / H3: 18px
- 正文: 16px / 辅助文字: 14px / 标签: 12px

### 3.4 布局规范

- **栅格系统**：12列网格，间距 16px / 24px / 32px
- **圆角**：卡片 12px，按钮 8px，标签 6px
- **阴影**：`0 4px 6px -1px rgba(0,0,0,0.3)` 微妙浮起感
- **动效**：hover 150ms ease-out，切换 300ms ease-in-out

### 3.5 分销佣金信息展示策略

| 位置 | 展示形式 | 内容 |
|------|---------|------|
| 首屏 Banner | 大字号渐变文字 | "推广赚佣金 · 最高 30% 返利" |
| API 卡片 | 金色标签 | "佣金 20%" |
| 详情页侧边栏 | 突出卡片 | 推广链接 + 预计收益计算器 |
| 用户中心 | 数据面板 | 累计收益/可提现/本月预估 |
| 推广中心 | 排行榜 | "月入过万推广员榜单"（社交证明） |
| 悬浮按钮 | 橙色渐变 | "推广赚 💰"（固定右下角） |

## 四、运营功能清单

### 4.1 推广链接管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 专属推广码 | 每个用户生成唯一推广码/链接 | P0 |
| API 级链接 | 不同 API 生成不同推广链接 | P0 |
| 短链接生成 | 美化短链接便于分享 | P1 |
| 二维码生成 | 推广链接转二维码（可下载） | P1 |
| 链接有效性检测 | 自动检测官方链接是否失效 | P2 |

### 4.2 佣金查看与提现

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 佣金明细 | 展示每笔佣金来源、时间、金额、状态 | P0 |
| 佣金计算器 | 输入预估点击量，计算预期收益 | P1 |
| 提现申请 | 支持微信/支付宝/银行卡 | P0 |
| 提现记录 | 展示历史提现记录及状态 | P0 |
| 佣金到账通知 | 公众号/短信/邮件通知 | P1 |
| 税务说明 | 佣金说明、税务提示 | P2 |

### 4.3 数据看板

| 看板指标 | 描述 | 更新频率 |
|---------|------|---------|
| 总点击量 | 推广链接总点击 | 实时 |
| 总订单数 | 通过链接产生的付费订单 | T+1 |
| 转化率 | 点击→付费转化率 | T+1 |
| 累计佣金 | 历史佣金总额 | 实时 |
| 可提现余额 | 当前可申请提现金额 | 实时 |
| 本月预估 | 本月预计佣金（含未确认） | 每日 |
| 趋势图 | 7天/30天/自定义时间范围趋势 | 每日 |
| 来源分析 | 各 API/渠道带来的流量和收益占比 | T+1 |

**数据看板图表建议：** 使用折线图（趋势）、饼图（来源占比）、柱状图（排名对比）

### 4.4 其他运营功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户等级体系 | 根据累计佣金划分推广员等级，等级越高佣金比例越高 | P1 |
| 佣金排行榜 | 月榜/总榜，展示 TOP 推广员（脱敏昵称） | P1 |
| 邀请奖励 | 推广员 A 邀请推广员 B，B 的佣金 A 可获额外 5% | P2 |
| 公告系统 | 佣金政策更新、API 上下架通知 | P0 |
| 工单系统 | 用户/推广员提交问题，运营后台处理 | P1 |
| 防作弊机制 | IP/设备指纹检测，异常佣金冻结 | P0 |

## 七、技术建议（可选，供开发团队参考）

- **前端**：Next.js / React + TailwindCSS（SSR 优化 SEO）
- **后端**：Node.js + PostgreSQL（或 Supabase）
- **搜索**：Algolia / Typesense（支持模糊搜索）
- **图表**：Recharts / Tremor
- **部署**：Vercel / Cloudflare Pages
- **追踪**：自建短链系统追踪跳转来源（避免依赖外部统计丢失数据）
- **支付**：佣金提现对接微信支付商家版 / 支付宝商家版
