# LLM API 聚合网站技术架构方案

## 一、技术架构方案

### 整体架构
采用**前后端分离**架构，前端 Next.js（App Router）+ 后端 Node.js/Express（或换 Go），数据库用 PostgreSQL + Redis，部署在 Vercel（前端）+ Railway/Render（后端），整体约 500 字说明如下。

### 核心模块拆分

| 模块 | 职责 | 技术选型 |
|------|------|---------|
| **网站前台** | LLM API 产品展示、筛选对比、工具下载页 | Next.js 15 + Tailwind CSS + shadcn/ui |
| **推广追踪系统** | 生成推广链接、记录点击/转化/佣金计算 | 后端 API + Redis 缓存 + UTM 参数解析 |
| **用户中心** | 注册登录、佣金余额、提现申请、订单历史 | NextAuth.js（或 Auth.js）做登录，后端 REST/GraphQL |
| **CMS 内容管理** | 后台管理 API 产品、工具链接、新闻公告 | Sanity CMS（或 Payload CMS 自托管） |
| **支付模块** | 对接微信/支付宝/PayPal，提现打款 | Stripe + 汇付/连连支付国内渠道 |
| **数据源同步** | 定时拉取 LLM 厂商定价、API 变更通知 | Cron Job + 厂商官方 API/网页爬取 |

### 推广链接追踪原理
用户注册 → 生成唯一推广码（如 `/ref/abc123`） → 链接携带 UTM 参数写入 Cookie → 被推广用户注册/付费后触发事件 → 佣金系统计算分成比例 → 记录到用户佣金账户。

### 数据流
```
用户点击推广链接 → CDN/中间层记录 PV → 注册转化 → 订单完成 → 佣金计算 → 提现审批 → 支付打款
```

## 二、MVP 功能优先级

### P0（必须上线）
1. **LLM API 产品展示** — 表格/卡片展示各厂商定价、模型、限速、状态
2. **用户注册/登录** — 邮箱 + 密码，推广码绑定
3. **推广链接生成** — 用户中心一键生成个人推广链接
4. **佣金记录** — 显示推广带来的订单、佣金比例、累计金额
5. **简单提现申请** — 用户发起提现，后台人工/半自动审批打款

### P1（第二版）
- AI 工具下载页汇集（分类、平台标签、官方链接）
- 佣金自动计算规则配置（按产品/按订单比例）
- 邮件通知（注册成功、佣金到账、提现进度）
- 管理员后台（产品管理、用户管理、订单审核）

### P2（后续迭代）
- 实时 API 定价同步（对接厂商 webhook 或定时爬取）
- 自动化提现（对接支付API直接打款）
- 佣金等级制度（拉新越多佣金比例越高）
- 多语言/国际化

## 三、技术栈推荐

### 前端（最快上线）
- **Next.js 15** (App Router) + **TypeScript** + **Tailwind CSS** + **shadcn/ui**
- 理由：全栈框架，自带 SSR/SEO 优化，生态庞大，模板众多，Vercel 一键部署
- 备选：Remix（更轻量）或 Nuxt（Vue 系）

### 后端
- **Node.js + Express** 或 **Hono**（更轻）
- 若追求性能：换 **Go + Gin** 或 **Rust + Actix**
- 理由：与前端同语言，生态成熟，开发速度快

### 数据库
- **PostgreSQL**（用户、订单、佣金记录等结构化数据）
- **Redis**（推广点击缓存、会话、限流）
- 理由：PostgreSQL 关系型数据强一致，佣金计算必须；Redis 支撑高并发推广PV

### CMS
- **Sanity CMS**（托管型，拖拽编辑，API 友好）
- 或 **Payload CMS**（自托管，TypeScript 原生）

### 支付
- **国内提现**：连连支付、汇付天下（需要企业资质）
- **海外/个人**：Stripe + PayPal
- **个人小微**：微信/支付宝当面付（个人可申请）

### 推广追踪
- 自建（PostgreSQL + Redis）即可满足初期
- 备选：Posthog（开源产品内分析，支持事件追踪）

### 部署
- 前端：Vercel（CDN 全球加速，免费额度充足）
- 后端：Railway / Render / 阿里云 ECS
- 数据库：Supabase（PostgreSQL托管）或 Railway
- 对象存储：Cloudflare R2 / 阿里云 OSS

## 五、关键风险与建议

1. **支付资质**：国内微信/支付宝提现需要企业账号，个人先上 Stripe + PayPal
2. **推广追踪精度**：Cookie 7天过期问题，建议用 sub1 归因窗口
3. **定价数据时效**：LLM 厂商频繁调价，建议建立数据源 CMS，运营人员可手动更新
4. **合规**：推广佣金需注意广告法，不要承诺"稳赚不赔"
5. **爬虫风险**：抓取厂商定价页面可能被封 IP，建议用官方公开 API 或手动维护
